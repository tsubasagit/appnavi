# UIビルダー評価レポート：中小企業経営者視点

## 評価対象
中小企業の経営者が、自社のスプレッドシートから顧客訪問登録アプリを簡単に作れるか？

## 評価結果：❌ **現状では困難**

### 致命的な問題点

#### 1. **スプレッドシートからの自動生成が不十分**
- **問題**: `createPageFromTemplate('spreadsheet')` が単純に列名をコピーするだけ
- **影響**: 
  - 「訪問日時」という列名が「テキスト入力」として生成される（本来は「日付・時刻」であるべき）
  - 「顧客名」「担当者」なども全て「テキスト入力」になり、適切なフィールドタイプが推測されない
  - 選択肢が必要な項目（例：「訪問結果」）が自動的に「選択肢」にならない

#### 2. **用語が専門的すぎる**
- **問題**: 「質問タイプ」「コンポーネント」「プロパティ」などの用語
- **影響**: 経営者には「何をすればいいか」が分からない
- **例**: 「質問タイプから追加」→ 経営者は「質問って何？」と混乱

#### 3. **ワークフローが不明確**
- **問題**: スプレッドシートをアップロード → UIタブ → テンプレート選択 → 生成、という流れが直感的でない
- **影響**: どこから始めればいいか分からない

#### 4. **データ構造とUIの対応が見えない**
- **問題**: スプレッドシートの列と生成されたフォームフィールドの対応が視覚的に分からない
- **影響**: 「このフィールドはどの列に対応しているの？」と混乱

#### 5. **フィールド編集が分かりにくい**
- **問題**: 右側のプロパティパネルが隠れていて、フィールドをクリックしないと表示されない
- **影響**: フィールド名を変更したり、必須設定を変更する方法が分からない

## 改善提案

### 優先度：高

1. **列名からフィールドタイプを自動推測**
   - 「日時」「日付」「訪問日」→ `datetime` または `date`
   - 「顧客名」「会社名」「企業名」→ `text`（必須推奨）
   - 「結果」「ステータス」「状態」→ `select`（選択肢を自動生成）
   - 「メモ」「備考」「コメント」→ `textarea`

2. **用語の改善**
   - 「質問タイプ」→ 「入力項目の種類」
   - 「コンポーネント」→ 「追加する項目」
   - 「プロパティ」→ 「設定」

3. **オンボーディングの強化**
   - 初回起動時に「スプレッドシートから自動生成」を推奨
   - 各ステップで「次に何をすればいいか」を明確に表示

4. **視覚的な対応関係の表示**
   - スプレッドシートの列名とフォームフィールドを並べて表示
   - 「このフィールドはスプレッドシートの『訪問日時』列に対応しています」と明示

5. **インライン編集の実装**
   - フィールド名を直接クリックして編集可能に
   - 必須チェックボックスをフィールドカード上に表示

### 優先度：中

6. **「顧客訪問登録」専用テンプレートの追加**
   - 一般的な訪問登録に必要なフィールドをプリセット
   - 経営者が「これだ！」と分かる選択肢

7. **プレビュー機能の強化**
   - 実際の入力画面をリアルタイムでプレビュー
   - 「これで正しいか？」を確認できる

8. **エラーメッセージの改善**
   - 技術的なエラーではなく、「この項目は必須です」など分かりやすいメッセージ

## 結論

現状のUIビルダーは、**技術者向け**の設計になっており、中小企業の経営者が直感的に使えるレベルには達していません。

特に「スプレッドシートから自動生成」機能が不十分で、生成されたフォームが実用的でない可能性が高いです。

**改善が必要な項目**:
1. 列名からのフィールドタイプ自動推測（最優先）
2. 用語の改善
3. オンボーディングの強化
4. 視覚的な対応関係の表示

これらの改善により、「スプレッドシートをアップロード → ボタン1つでフォーム生成 → 微調整」という理想的なワークフローが実現できます。
