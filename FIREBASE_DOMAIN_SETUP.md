# Firebase 認証済みドメイン設定ガイド

## エラーが発生した場合

Google認証で「このドメインはFirebaseの認証済みドメインリストに登録されていません」というエラーが表示された場合、以下の手順でドメインを追加してください。

## 設定手順

### 1. Firebase Consoleにアクセス
1. [Firebase Console](https://console.firebase.google.com) にアクセス
2. プロジェクト「**appnavi-add7e**」を選択

### 2. 認証設定を開く
1. 左側のメニューから「**認証**」をクリック
2. 上部のタブから「**設定**」を選択
3. 「**承認済みドメイン**」タブを開く

### 3. ドメインを追加
「**ドメインを追加**」ボタンをクリックして、以下のドメインを追加してください：

#### 必須ドメイン
- **`tsubasagit.github.io`** - GitHub Pagesの本番環境
- **`localhost`** - ローカル開発環境

#### その他のドメイン（必要に応じて）
- カスタムドメインを使用している場合は、そのドメインも追加してください
- 例: `appnavi.example.com`

### 4. 設定の反映
- ドメインを追加すると、すぐに反映されます
- ページを再読み込みして、再度Google認証を試してください

## 注意事項

- ドメイン名は正確に入力してください（`http://`や`https://`は不要）
- サブドメインも個別に追加する必要があります
- デフォルトで`localhost`は含まれている場合がありますが、明示的に追加することを推奨します

## トラブルシューティング

### エラーが続く場合
1. ブラウザのキャッシュをクリア
2. 数分待ってから再度試す（反映に時間がかかる場合があります）
3. Firebase Consoleでドメインが正しく追加されているか確認

### 開発環境での確認
- ローカル開発時は`localhost`が追加されていることを確認してください
- ポート番号付きのURL（例: `localhost:8080`）を使用している場合は、`localhost`のみで動作します
