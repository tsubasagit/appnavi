# AppNavi 仕様書

## 1. システム概要

### 1.1 目的

AppNaviは、Excel/Google Sheetsで管理しているデータから、コードを書かずにWebアプリケーションを作成できるプラットフォームです。

### 1.2 対象ユーザー

- 中小企業の経営者（IT初心者）
- 地方自治体の職員（DX推進担当）
- 非営利団体（データ管理者）
- スプレッドシートでデータ管理している個人・組織

### 1.3 主要機能

1. **データ管理**: Excel/CSVからのデータインポート、テーブル表示、編集、削除
2. **UIビルダー**: フォーム作成、ページ管理、テンプレート選択
3. **データ可視化**: グラフ生成、集計、分析
4. **スプレッドシート連携**: Google Sheetsとのリアルタイム同期
5. **エクスポート**: CSV/Excel形式でのデータエクスポート

## 2. 機能仕様

### 2.1 認証・ユーザー管理

#### 2.1.1 AppNaviIDシステム

**必須要件:**
- すべての機能でAppNaviIDが必須
- AppNaviID未設定時は自動的に登録ページにリダイレクト

**AppNaviID形式:**
- UUID v4形式: `xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx`
- 正規表現: `/^[a-zA-Z0-9]{8}-[a-zA-Z0-9]{4}-[a-zA-Z0-9]{4}-[a-zA-Z0-9]{4}-[a-zA-Z0-9]{12}$/`

**機能:**
- 新規発行: ランダムにUUID v4を生成
- 既存ID入力: ユーザーが既存のAppNaviIDを入力
- IDコピー: クリップボードにコピー
- ID変更: 新しいIDを発行（既存データは引き継がれない）

**保存場所:**
- localStorage: `appnavi_user_id`

#### 2.1.2 ユーザーデータ分離

- すべてのデータ操作でAppNaviIDでフィルタリング
- ユーザーは自分のデータのみ表示・編集可能
- Firestore、localStorageともにAppNaviIDで分離

**設定項目:**

## 3. データモデル

### 3.1 アプリデータ

```javascript
{
  id: string,                    // アプリID（自動生成）
  name: string,                  // アプリ名
  description: string,           // 説明
  userId: string,                // AppNaviID（必須）
  columns: Array<Column>,        // 列定義
  data: Array<Row>,              // データ行
  pages: Array<Page>,            // UIビルダーのページ
  createdAt: timestamp,          // 作成日時
  updatedAt: timestamp,          // 更新日時
  storageType: 'local' | 'firebase',  // 保存方法
  googleSheetsUrl: string,       // Google Sheets URL（オプション）
  googleAccessToken: string,      // Google認証トークン（オプション）
  googleSheetsLastSynced: timestamp  // 最終同期時刻（オプション）
}
```

### 3.2 列定義（Column）

```javascript
{
  name: string,                  // 列名
  type: 'text' | 'number' | 'date' | 'datetime' | 'select',  // データ型
  required: boolean,            // 必須/任意
  options: Array<string>         // 選択肢（select型の場合）
}
```

### 3.3 データ行（Row）

```javascript
{
  [columnName: string]: any      // 列名をキーとしたオブジェクト
}
```

### 3.4 ページ定義（Page）

```javascript
{
  id: string,                    // ページID
  name: string,                  // ページ名
  fields: Array<Field>           // フィールド定義
}
```

### 3.5 フィールド定義（Field）

```javascript
{
  type: 'text' | 'textarea' | 'select' | 'radio' | 'checkbox' | 'date' | 'number',
  label: string,                 // ラベル名
  required: boolean,             // 必須/任意
  options: Array<string>         // 選択肢（select/radio/checkboxの場合）
}
```

## 4. API仕様

### 4.1 Google Sheets API

**認証:**
- OAuth 2.0（Google Identity Services）
- アクセストークンはlocalStorageに保存

**エンドポイント:**
- 読み込み: `GET https://sheets.googleapis.com/v4/spreadsheets/{spreadsheetId}/values/{range}`
- 書き込み: `POST https://sheets.googleapis.com/v4/spreadsheets/{spreadsheetId}/values/{range}:append`
- 更新: `PUT https://sheets.googleapis.com/v4/spreadsheets/{spreadsheetId}/values/{range}`
- 削除: `DELETE https://sheets.googleapis.com/v4/spreadsheets/{spreadsheetId}/values/{range}:clear`

**エラーハンドリング:**
- 401: 認証エラー → トークンリフレッシュ
- 403: 権限エラー → ユーザーに解決方法を提示
- 400: リクエストエラー → エラーメッセージを表示

### 4.2 Firebase Firestore

**コレクション:**
- `apps`: アプリデータ

**セキュリティルール:**
- ユーザーは自分のデータ（userId == AppNaviID）のみアクセス可能

**クエリ:**
- アプリ一覧: `where('userId', '==', appNaviID).orderBy('createdAt', 'desc')`
- アプリ取得: `doc(appId)` + userIdチェック

## 6. パフォーマンス要件

### 6.1 読み込み時間

- 初回読み込み: 3秒以内
- ページ遷移: 1秒以内
- データ読み込み: 2秒以内（1000件まで）

### 6.2 レスポンス時間

- データ保存: 500ms以内（localStorage）
- データ同期: 2秒以内（Google Sheets）
- UI更新: 100ms以内

### 6.3 データ容量

- ローカルストレージ: 5MB制限（ブラウザ依存）
- Firestore: 無料枠内（1GBまで）

## 7. セキュリティ要件

### 7.1 認証

- AppNaviID必須
- AppNaviID形式検証
- 不正なAppNaviIDの拒否

### 7.2 データアクセス

- ユーザーは自分のデータのみアクセス可能
- AppNaviIDでデータを完全分離
- Firestoreセキュリティルールで保護

### 7.3 データ保護

- データ暗号化: HTTPS必須
- トークン保護: localStorageに保存（同一オリジンのみアクセス）
- XSS対策: 入力値のサニタイズ

## 8. エラーハンドリング

### 8.1 エラー種類

**認証エラー:**
- AppNaviID未設定 → 登録ページにリダイレクト
- AppNaviID形式不正 → エラーメッセージ表示

**データエラー:**
- データ読み込み失敗 → エラーメッセージ表示、localStorageからフォールバック
- データ保存失敗 → エラーメッセージ表示、リトライ可能

**APIエラー:**
- Google Sheets APIエラー → 詳細なエラーメッセージと解決方法を表示
- Firebaseエラー → エラーメッセージ表示、localStorageにフォールバック

### 8.2 エラーメッセージ

- **ユーザーフレンドリー**: 技術用語を避ける
- **解決方法を提示**: エラーの解決手順を表示
- **日本語**: すべてのメッセージを日本語で表示

## 10. 制約事項

### 10.1 技術的制約

- **ブラウザ依存**: モダンブラウザのみ対応
- **localStorage制限**: 5MB制限（ブラウザ依存）
- **Firestore制限**: 無料枠内での利用を想定

### 10.2 機能的制約

- **セル書式**: Excelのセル書式は非対応（データのみ）
- **関数**: Excel関数は非対応（データのみ）
- **複数シート**: 1シートのみ対応（将来対応予定）

### 10.3 セキュリティ制約

- **AppNaviID必須**: すべての機能で必須
- **データ分離**: ユーザーは自分のデータのみアクセス可能
- **公開コード**: 機密情報を含まない設計
