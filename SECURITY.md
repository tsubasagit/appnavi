# セキュリティポリシー

## セキュリティに関する報告

セキュリティ上の問題を発見した場合は、GitHubのセキュリティアドバイザリ機能を使用して報告してください。

## OAuth 2.0 クライアントIDについて

### クライアントIDは公開しても安全です

AppNaviはブラウザベースのアプリケーションであり、OAuth 2.0のクライアントIDを使用します。

**重要なポイント**:

1. **クライアントID（Client ID）**: 
   - ✅ 公開しても安全です
   - ブラウザのJavaScriptコードに含まれ、誰でも見ることができます
   - OAuth 2.0の仕様上、公開されることを前提に設計されています

2. **クライアントシークレット（Client Secret）**:
   - ❌ 絶対に公開してはいけません
   - サーバーサイドアプリケーションでのみ使用されます
   - AppNaviはブラウザベースのアプリなので、クライアントシークレットは使用しません

### セキュリティのベストプラクティス

#### デモ用クライアントIDを提供する場合

1. **承認済みのJavaScript生成元を制限**:
   - デモサイトのURLのみを許可
   - 例: `https://tsubasagit.github.io` のみ

2. **スコープを最小限に設定**:
   - 必要な権限のみを要求
   - 例: `https://www.googleapis.com/auth/spreadsheets`

3. **使用量制限を設定**:
   - Google Cloud ConsoleでAPI使用量の制限を設定
   - 不正使用を防ぐ

#### ユーザーが自分のクライアントIDを設定する場合（推奨）

1. **各ユーザーが自分のクライアントIDを作成**:
   - Google Cloud Consoleでプロジェクトを作成
   - 自分のドメインで使用するクライアントIDを生成

2. **設定画面で簡単に設定**:
   - AppNaviの設定画面からクライアントIDを入力
   - `localStorage`に保存（ブラウザローカル）

3. **クライアントIDは公開されても問題なし**:
   - ユーザーのブラウザに保存される
   - 他のユーザーには影響しない

### なぜクライアントIDは安全なのか？

OAuth 2.0の仕組みでは、クライアントIDだけでは以下のことができません：

1. **他のユーザーのデータにアクセスできない**:
   - 各ユーザーが自分のGoogleアカウントで認証する必要がある
   - クライアントIDだけでは認証できない

2. **クライアントシークレットなしではトークンを取得できない**:
   - ブラウザベースのアプリでは、ユーザーが直接Googleに認証する
   - サーバーを介さないため、クライアントシークレットは不要

3. **承認済みの生成元でしか動作しない**:
   - Google Cloud Consoleで設定したURLからのみ認証が可能
   - 悪意のあるサイトでは使用できない

### 実際のセキュリティリスク

**低リスク**:
- クライアントIDが公開されること（これは正常な動作）

**中リスク**:
- 承認済みのJavaScript生成元が広すぎる設定
- スコープが広すぎる設定

**高リスク**:
- クライアントシークレットの漏洩（AppNaviでは使用しない）
- 承認済みのリダイレクトURIが広すぎる設定

### 推奨事項

1. **オープンソースプロジェクトとして公開する場合**:
   - デモ用のクライアントIDを提供しても良い（適切に制限を設定）
   - ユーザーが自分のクライアントIDを設定できるようにする（推奨）
   - READMEに設定方法を記載する

2. **本番環境で使用する場合**:
   - 必ず自分のクライアントIDを作成して使用する
   - 承認済みのJavaScript生成元を自分のドメインのみに制限
   - 定期的に使用状況を確認する

## データの保存について

### Firebase Firestore

- データはFirebase Firestoreに保存されます
- セキュリティルールを適切に設定してください
- 本番環境では認証を必須にすることを推奨します

### LocalStorage

- 開発時やデモ用に使用されます
- ブラウザローカルに保存されるため、他のユーザーには見えません
- 本番環境ではFirebaseの使用を推奨します

### Googleスプレッドシート

- ユーザーが指定したスプレッドシートにデータを保存します
- スプレッドシートの共有設定はユーザーが管理します
- AppNaviはユーザーの認証情報を使用してアクセスします

## その他のセキュリティ考慮事項

1. **HTTPSの使用**:
   - GitHub Pagesは自動的にHTTPSを提供します
   - 本番環境では必ずHTTPSを使用してください

2. **CSP（Content Security Policy）**:
   - 必要に応じてCSPヘッダーを設定してください

3. **XSS対策**:
   - ユーザー入力は適切にエスケープしてください
   - 現在の実装では、データは適切に処理されています

4. **CSRF対策**:
   - OAuth 2.0の認証フローはCSRF対策が組み込まれています
