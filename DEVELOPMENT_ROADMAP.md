- [ ] `writeToSpreadsheet`メソッドの動作確認
  - 新規行の追加

- [ ] データタブに編集ボタンを追加

- [ ] 認証エラーの適切な処理

## 🔄 ブランチ戦略

### ブランチ命名規則
- `main`: 本番環境用（安定版）
- `v1-spreadsheet-integration`: V1機能のバックアップ
- `feature/[機能名]`: 新機能開発用
- `fix/[バグ名]`: バグ修正用
- `hotfix/[緊急修正名]`: 緊急修正用

### 開発フロー
1. `main`から`feature/[機能名]`ブランチを作成
2. 機能を実装
3. テストとレビュー
4. `main`にマージ
5. GitHub Pagesに自動デプロイ
