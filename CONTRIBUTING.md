## 開発環境のセットアップ

### 前提条件

- Node.js（オプション）
- Python 3（オプション）
- モダンブラウザ

### セットアップ手順

1. **リポジトリのフォーク**
   ```bash
   git clone https://github.com/tsubasagit/appnavi.git
   cd appnavi
   ```

2. **ローカルサーバーの起動**
   ```bash
   # Windows
   start-local-server.bat
   
   # Mac/Linux
   python3 -m http.server 8000
   ```

3. **ブラウザでアクセス**
   ```
   http://localhost:8000
   ```

詳細は[DEVELOPMENT_GUIDE.md](./DEVELOPMENT_GUIDE.md)を参照してください。

## コーディング規約

### JavaScript

- **ES6+**: モダンなJavaScriptを使用
- **Vanilla JS**: フレームワーク不使用
- **命名規則**: camelCase（変数・関数）、PascalCase（クラス）
- **コメント**: 日本語で記述

### CSS

- **Tailwind CSS**: ユーティリティファースト
- **カスタムCSS**: 最小限に

### ファイル構造

```
appnavi/
├── index.html          # ダッシュボード
├── myapp.html          # アプリページ
├── login.html          # ログインページ
├── signup.html         # サインアップページ
├── appnavi-id-register.html  # AppNaviID登録ページ
├── firebase-config.js  # Firebase設定
├── google-sheets-api.js # Google Sheets API
├── appnavi-id.js       # AppNaviID管理
└── docs/               # ドキュメント
```

## プルリクエストの手順

### 1. ブランチの作成

```bash
git checkout -b feature/your-feature-name
# または
git checkout -b fix/your-bug-fix
```

### 2. 変更の実装

- 機能を実装
- テストを追加（可能な場合）
- ドキュメントを更新

### 3. コミット

```bash
git add .
git commit -m "feat: 新機能の説明"
# または
git commit -m "fix: バグ修正の説明"
```

**コミットメッセージの形式:**
- `feat:` 新機能
- `fix:` バグ修正
- `docs:` ドキュメント
- `style:` コードスタイル
- `refactor:` リファクタリング
- `test:` テスト
- `chore:` その他

### 4. プッシュ

```bash
git push origin feature/your-feature-name
```

### 5. プルリクエストの作成

1. GitHubでプルリクエストを作成
2. 変更内容を説明
3. 関連するIssueがあればリンク
4. レビューを待つ
